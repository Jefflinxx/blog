# GitHub Pages 部署流程

這份文件記錄本專案部署到 GitHub Pages 的方式。

## 部署目標

GitHub repo：

```text
https://github.com/Jefflinxx/blog
```

GitHub Pages 網址：

```text
https://jefflinxx.github.io/blog/
```

## 部署原理

這個 blog 是靜態網站。部署流程是：

```text
Markdown 文章
  -> npm run seo:check
  -> npm run content-map:generate
  -> npm run sitemap:generate
  -> mkdocs build
  -> 產生 site/
  -> GitHub Actions 上傳到 GitHub Pages
```

真正被部署的是 `site/` 裡面的 HTML/CSS/JS 靜態檔案。

## 重要檔案

GitHub Pages workflow：

```text
.github/workflows/deploy-pages.yml
```

網站設定：

```text
mkdocs.yml
```

部署前檢查工具：

```text
scripts/seo-check.mjs
scripts/generate-content-map.mjs
scripts/generate-sitemap.mjs
```

## 本機發布前檢查

每次要部署前，建議先在本機跑：

```bash
npm run seo:check
npm run content-map:generate
npm run crosslink:suggest
npm run sitemap:generate
npm run build
```

如果以上都通過，再 commit 和 push。

## 建立新文章後的流程

建立新文章：

```bash
npm run post:new -- "文章標題"
```

寫完文章後，確認 frontmatter：

```yaml
---
title: "文章標題"
date: 2026-05-24
description: "一段適合搜尋結果和文章列表顯示的摘要。"
categories:
  - Notes
tags:
  - blog
authors:
  - you
draft: false
---
```

發布前確認：

- `draft: false`
- `description` 不為空
- `date` 是 `YYYY-MM-DD`
- 有放 `<!-- more -->`
- 內部連結是有意義的，不是硬塞

## Push 觸發部署

GitHub Actions 設定為：只要 push 到 `main` 分支，就會自動部署。

一般流程：

```bash
git status
git add .
git commit -m "Update blog"
git push origin main
```

push 後，到 GitHub repo 的 Actions 頁面查看部署狀態：

```text
https://github.com/Jefflinxx/blog/actions
```

成功後，網站會出現在：

```text
https://jefflinxx.github.io/blog/
```

## GitHub Pages 設定

這個專案使用 GitHub Actions 部署 Pages。

在 GitHub 網頁上可以檢查：

```text
Repo -> Settings -> Pages
```

Source 應該選：

```text
GitHub Actions
```

如果不是，請改成 GitHub Actions。

## 常見問題

### 1. GitHub Pages 顯示 404

可能原因：

- GitHub Actions 還沒跑完
- Pages source 沒有設成 GitHub Actions
- `site_url` 設錯
- repo 還沒有啟用 Pages

先看 Actions：

```text
https://github.com/Jefflinxx/blog/actions
```

再看 Pages 設定：

```text
https://github.com/Jefflinxx/blog/settings/pages
```

### 2. 文章沒有出現在網站

確認文章是否放在：

```text
docs/writing/posts/
```

確認 frontmatter 是否有：

```yaml
draft: false
```

再跑：

```bash
npm run seo:check
npm run build
```

### 3. Sitemap 或 RSS 網址不對

檢查 `mkdocs.yml`：

```yaml
site_url: https://jefflinxx.github.io/blog/
```

檢查 `docs/robots.txt`：

```text
Sitemap: https://jefflinxx.github.io/blog/sitemap.xml
```

改完後重新產生 sitemap：

```bash
npm run sitemap:generate
```

### 4. GitHub Actions build 失敗

先在本機跑同樣的檢查：

```bash
npm run seo:check
npm run content-map:generate
npm run sitemap:generate
npm run build
```

如果本機成功，但 GitHub Actions 失敗，就看 Actions log 裡是哪一步失敗。

## 手動重新部署

如果沒有新的 commit，但想重新部署，可以到：

```text
https://github.com/Jefflinxx/blog/actions
```

選擇 `Deploy Pages` workflow，按 `Run workflow`。
